# AI Restaurant Recommendation Service - Architecture

## 1. Project Overview
A full-stack, AI-driven restaurant recommendation system that leverages a dataset of 51,000+ restaurants and the Groq (Llama 3.3) LLM to provide personalized dining suggestions.

---

## 2. Technology Stack

### Frontend
- **Framework**: Next.js 15 (App Router, TypeScript)
- **Styling**: Tailwind CSS (Glassmorphic Design)
- **Animations**: Framer Motion
- **Icons**: Lucide React
- **HTTP Client**: Axios

### Backend
- **Framework**: FastAPI (Python 3.9+)
- **Data Processing**: Pandas
- **Dataset**: Zomato Restaurant Dataset (Hugging Face)
- **AI Integration**: Groq SDK (Llama-3.3-70b-versatile)
- **Server**: Uvicorn

---

## 3. System Data Flow

1.  **Request**: User selects filters (Area, Cuisine, Price Range, Rating, Count) on the Next.js frontend.
2.  **Filtering**: FastAPI receives the POST request. The `DataManager` (Pandas) filters the 51,000 records down to $N$ candidates based on hard constraints.
3.  **Ranking**: Candidates are ranked by a hybrid score: `Rating` (primary) and `Avg Cost` (secondary).
4.  **AI Rationale**: The top candidates' names and user context are hashed and sent to **Groq LLM**. The LLM generates a natural language explanation for why these specific restaurants match the user's intent.
5.  **Deep-Dive**: If a user clicks "View AI Deep-Dive", a separate API call triggers a second LLM chain that analyzes specific reviews and liked dishes for that specific restaurant to generate Pros/Cons.
6.  **Response**: The frontend renders the results with animated glass cards and AI-generated insights.

---

## 4. Implementation Phases

### Phase 1: Data Infrastructure
- **Focus**: Loading and cleaning.
- **Key Logic**: Standardized messy rating strings (e.g., `4.1/5`, `NEW`) and cost strings into numeric types using Regex.
- **Storage**: In-memory Pandas DataFrame for low-latency search.

### Phase 2: AI Core & Initial API
- **Focus**: Connectivity.
- **Key Logic**: Integrated `groq-python` and established the first `/recommend` endpoint.
- **Security**: Environment variables (`.env`) for API keys.

### Phase 3: Premium UI
- **Focus**: UX/UI.
- **Key Logic**: Built a responsive dark-theme dashboard with loading states and Framer Motion transitions.

### Phase 4: Optimization & Refinement (Final)
- **Focus**: Performance & Detail.
- **Key Logic**: 
    - **Caching**: Added `lru_cache` to Groq rationale generation to save LLM tokens and time.
    - **Advanced Search**: Added Price-range sliders with logical guards (`min <= max`).
    - **Deep-Dive API**: Created `/restaurant/{name}/details` for per-restaurant AI analysis.
    - **Metadata API**: Created `/metadata` to dynamically populate frontend dropdowns from the unique values in the dataset.

---

## 5. API Documentation

### `GET /metadata`
- **Output**: Returns unique list of `areas` and `cuisines` found in the Zomato dataset.

### `POST /recommend`
- **Input**: `{ area, min_cost, max_cost, min_rating, cuisine, top_k }`
- **Output**: Mentions `candidates` (List) and `ai_rationale` (String).

### `GET /restaurant/{name}/details`
- **Output**: Returns `metadata` for the restaurant and an `ai_insight` (Pros/Cons/Verdict) generated by Llama 3.3.
